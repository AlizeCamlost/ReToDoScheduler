# 恢复流程

相关文档：

- [server-deploy.md](server-deploy.md)

## 1. 本地恢复

1. 停止 app 进程
2. 恢复最新本地压缩快照
3. 重新打开 app
4. 验证任务总量、完成状态与最近修改是否符合预期

## 2. 远端恢复

1. 准备临时 PostgreSQL 实例
2. 恢复最近一次可用 dump
3. 如存在操作日志，再回放快照时间点之后的同步操作
4. 将 API 指向恢复后的数据库
5. 重新验证健康检查和任务校验

## 3. 校验清单

- 任务总数和完成数在合理范围内
- 最近更新的任务仍存在
- 关键任务的 DDL、状态和文本内容无缺失
- 调度块内部无明显时间冲突

## 4. 恢复后的动作

- 先做只读校验，再恢复客户端写流量
- 如果恢复来源于生产事故，记录恢复时间点、使用的备份、丢失窗口和后续补救动作
