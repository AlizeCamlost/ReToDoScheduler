# 恢复流程

相关文档：

- [server-deploy.md](server-deploy.md)

## 1. 总原则

恢复流程先停写，再恢复，再校验，最后恢复写流量。

## 2. 本地恢复

1. 停止 app 进程
2. 恢复最近一次可用的本地快照
3. 重新打开 app
4. 做只读校验，确认任务数量、完成状态和最近修改符合预期

## 3. 远端恢复

1. 准备临时 PostgreSQL 实例或隔离恢复环境
2. 恢复最近一次可用 dump
3. 如有额外操作日志，再回放快照时间点之后的变更
4. 将 API 切到恢复后的数据库
5. 重新验证 `/health` 和关键业务数据

## 4. 校验清单

- 任务总数和完成数在合理范围内
- 最近更新的任务仍存在
- 关键任务的 DDL、状态和文本内容无缺失
- 调度视图没有明显时间冲突或异常空洞

## 5. 恢复后的动作

- 先做只读校验，再恢复客户端写流量
- 记录恢复时间点、使用备份、丢失窗口和补救动作
- 如果是生产事故，补充事后复盘
